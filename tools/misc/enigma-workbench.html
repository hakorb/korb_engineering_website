How it works

This is a faithful software implementation of the German Enigma cipher machine, extended to 24 rotor options (historical + workbench extras) so you can design configurations the Wehrmacht never built. Every encryption is reversible: if the recipient configures an identical machine with the same starting positions, they decrypt by typing the ciphertext.

Quick start

1. Pick 3 or 4 rotors from the dropdowns. Historical rotors are listed first: I–V (Wehrmacht), VI–VIII (Kriegsmarine), Beta & Gamma (M4 thin), K-I / K-II / K-III (Swiss-K). IX–XXIII are unique workbench rotors.
2. Set POS (the letter showing in each rotor window) and optional RING (internal wiring shift). These are your daily key.
3. Choose a reflector (UKW-B & UKW-C are historical; Thin-B / Thin-C are for 4-rotor mode; Reichsbahn-A is older).
4. Optionally wire the plugboard — tap one letter, then another, to create a swap pair (max 10 pairs).
5. Type or paste text into Plaintext. The ciphertext appears live in the right panel. Press ⧉ Copy with Settings to get a text blob you can paste into a message; the recipient pastes it into their workbench, hits ⇩ Import Config String, and decrypts.

Config string format

A config string looks like ENGMA1|4|VIII,III,II,Beta|QWER|AAAA|thinB|AB CD EF GH — version, rotor count, rotors (left→right), positions, rings, reflector, plugboard pairs. It's URL-safe so you can DM it, email it, or put it in a git commit.

Notes & caveats

Only A–Z are encrypted. By default non-letters are stripped (digits can optionally be spelled out). With Preserve spaces & punctuation on, whitespace and punctuation pass through for readability, but the letter encryption remains identical. The machine is historically accurate for stepping, including the double-step anomaly on 3-rotor mode. Enigma was broken by Bletchley Park in the 1940s — do not use this for real secrets. It's a fun classroom-grade cipher and a beautiful mechanical simulation, not modern cryptography.
