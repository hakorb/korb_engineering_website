Data Management

Raw Storage Keys

All data lives under korb_deal_hunter_* in localStorage.

  • korb_deal_hunter_library — items array
  • korb_deal_hunter_settings — currency rates & prefs
  • korb_deal_hunter_version — schema version

How It Works

Tracking Items

Add items via the Add Item tab or the Catalog tab. Each item stores MSRP, current price, target price, store, brand, category, tags, and notes.

When current < target, the card gets a green DEAL badge and moves to the "On Sale" kanban column automatically.

Price History & Charts

Each manual price update appends to the item's price history. Click "History" on any card to see the trend line (pure Canvas, no libraries) and the raw price log.

The chart y-axis auto-scales; x-axis shows chronological update order.

Search & Product Links

Every card exposes a Google Shopping search link for the item name. If you provided a direct product URL, a second Open Product Page link appears.

Kanban Backlog

Four columns: Wishlist → On Sale → Purchased → Returned/Done. Drag on desktop or use the ► buttons on cards. Great for managing your shopping pipeline.

Currency Conversion

Set your base display currency and manual exchange rates in Settings. Since stores don't offer free rate APIs, you enter rates manually — update them when you notice drift.

All prices normalize internally to USD and display in the selected base.

Import / Export

Export produces a JSON file of your entire library + settings. Import merges or replaces (you'll be asked). Great for device-to-device moves or backups.

Disclaimer

This is a personal tracker. It never contacts any store APIs or scrapes prices — you update prices manually when you spot deals. Catalog MSRPs are placeholder defaults and may differ from actual store pricing.
